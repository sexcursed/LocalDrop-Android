# LocalDrop 📱⚡📱

**LocalDrop** — это легковесное Android-приложение для обмена сообщениями и данными в локальной сети по протоколу P2P (Peer-to-Peer). Проект разработан с использованием современного стека технологий Jetpack Compose и асинхронных сетевых интерфейсов, являясь производительным и приватным аналогом таких решений, как LocalSend или Snapdrop.

Приложение автоматически обнаруживает устройства в общей Wi-Fi сети без использования внешних серверов или интернета, обеспечивая максимальную конфиденциальность передаваемых данных.

---

## 🛠 Стек технологий и библиотеки

* **Language:** [Kotlin](https://kotlinlang.org/) (100%) — современный и безопасный язык разработки.
* **UI Слой:** [Jetpack Compose](https://developer.android.com/jetpack/compose) — декларативный фреймворк для построения интуитивного реактивного интерфейса.
* **Архитектура:** Clean Architecture + MVVM + UDF (Unidirectional Data Flow) — строгое разделение ответственности на независимые слои (Data, Domain, Presentation).
* **Асинхронность:** [Kotlin Coroutines](https://github.com/Kotlin/kotlinx.coroutines) & [SharedFlow / StateFlow](https://kotlinlang.org/api/kotlinx.coroutines/kotlinx-coroutines-core/kotlinx.coroutines.flow/-state-flow/) — реактивное управление потоками данных.
* **Network Discovery:** Android `NsdManager` (Network Service Discovery / mDNS) — протокол обнаружения сервисов для динамического поиска активных узлов.
* **Сетевой транспорт:** Низкоуровневые Java/Kotlin TCP-сокеты (`ServerSocket`, `Socket`) для стабильного и контролируемого P2P-соединения.
* **Dependency Injection:** [Koin](https://insert-koin.io/) — легковесный и гибкий контейнер внедрения зависимостей, оптимизированный под Jetpack Compose.

## 📐 Архитектура проекта

Проект построен по принципам **Clean Architecture**, что изолирует бизнес-логику от деталей реализации (UI и фреймворков):

[ Презентационный слой (UI / Compose) ]
                  │
                  ▼ (Вызов UseCase)
[ Доменный слой (Domain / Бизнес-логика) ]             <───           [ Модели (NetworkDevice / Message) ]
                  ▲
                  │ (Реализация интерфейсов репозиториев)
[ Слой данных (Data / Сеть / NSD / Sockets) ]

### Основные компоненты и структура файлов:
* **`domain/`** (Чистая бизнес-логика):
  * `model/`: Неизменяемые структуры данных (`NetworkDevice`, `TransferMessage`).
  * `repository/`: Интерфейсы репозиториев, определяющие контракт для работы с сетью (`NetworkDiscoveryRepository`, `P2pConnectionRepository`).
  * `usecase/`: Атомарные сценарии использования (`StartDiscoveryUseCase`, `StartBroadcastingUseCase`, `StartServerUseCase`, `SendMessageUseCase`, `StopBroadcastingUseCase`).
* **`data/`** (Инфраструктурный слой):
  * `NsdDiscoveryImpl`: Реализует поиск устройств через системный `NsdManager` с использованием `callbackFlow` для реактивной передачи списков активных пиров. Работает по сетевому идентификатору `_localDrop._tcp.`.
  * `P2pConnectionImpl`: Отвечает за сокетное соединение. Поднимает TCP-сервер на фиксированном порту `7777` и обрабатывает низкоуровневый ввод-вывод потоков байт (`DataInputStream` / `DataOutputStream`).
* **`presentation/`** (Слой представления):
  * `MainViewModel`: Управляет жизненным циклом компонентов, агрегирует данные из UseCase и транслирует единый стейт. Автоматически генерирует UUID устройства при инициализации.
  * `DiscoveryUiState`: Насыщенный иммутабельный стейт интерфейса (хранит состояние сканирования, вещания, список найденных устройств и историю сообщений).
* **`di/` & `ui/`** (Запуск и конфигурация):
  * `appModule.kt`: Описание графа зависимостей с разделением на `single` для репозиториев/менеджеров и `factory` / `viewModel` для динамических объектов.

## 🚦 Текущее состояние проекта

На данный момент в репозитории реализован базовый рабочий каркас приложения:
1. **Регистрация в сети:** Приложение успешно регистрирует устройство в локальной службе mDNS под уникальным именем, содержащим сгенерированный UUID.
2. **Сканирование:** Настроен асинхронный поиск других устройств в локальном Wi-Fi пространстве с выводом списка в реактивный UI-компонент (`LazyColumn`).
